🛡️ Совет по кибербезопасности

Закон требует не «российский логин», а российский контроль над самим сервисом авторизации. Поэтому выбор между провайдерами — это выбор по трём осям: юридическое соответствие (в реестре или нет), скорость интеграции и охват. Для массового портала стандартный OAuth 2.0 от Яндекс ID или VK ID закрывается за день; банковские ID (Сбер, T-ID, Альфа) потребуют оферту и модерацию. Для КИИ и госзначимых действий нужен ЕСИА либо тяжёлый IdM класса Avanpost и Solar с сертификатами ФСТЭК.

Отдельно про пользователей, которые заходили только через Google: при отключении иностранного входа их доступ и данные рискуют потеряться. Продумайте связывание аккаунтов по номеру телефона до переключения, а не после. Telegram-вход рассматривайте как зону риска наравне с Google — авторизатором фактически выступает иностранный мессенджер.

Столбец «Сложность интеграции» — сколько сил уйдёт на подключение

  • 🟢 Просто Есть документация, всё делается онлайн: регистрируете приложение, получаете ключи, вставляете кнопку. Ни договоров, ни поездок в офис.
  • 🟡 Средне Надо напрячься с договором или офертой: заявка менеджеру, подписание по ЭДО, выпуск сертификата, модерация. Протоколы стандартные.
  • 🟠 Сложно Закрытый контур плюс русская криптография: подпись через КЭП/УКЭП, mTLS, согласования с ИБ провайдера. Интеграционный проект.
  • 🔴 Очень сложно ГОСТ-HTTPS, своя PKI и СКЗИ, биометрия или требования Минцифры/ФСБ. Оценка влияния на СКЗИ — месяцы, при своей разработке до 1,5–3 лет.

Столбец «Интеграция» — тип подключения

  • 🔓 OAuth Открытый стандарт, публичное API — чужой сайт может подключиться сам.
  • 🔒 Internal Закрытый или проприетарный протокол, доступ по договору или в составе продукта.

Столбец «Реестр» — юридический шлюз

  • ✅ В реестре Решение в реестре российского ПО — доказуемое соответствие требованиям к защите информации.
  • — В реестре нет либо это внешний сервис входа: соответствие держится на российском владельце сервиса.

Как проверялась сложность (обновление 06.07.2026)

  • Оценки сверены с официальной документацией провайдеров: developers.sber.ru, developer.tbank.ru, id.vk.com, yandex.ru/dev, технологический портал ЕСИА и методические рекомендации Минцифры (обновлены август 2025).
  • Ключевой вывод: реально «встроить за день без договора» можно лишь 7 из 67 систем. Все банковские ID (Сбер, T‑ID, Альфа и прочие) требуют оферты по ЭДО, сертификата и модерации — это 🟡, а не 🟢.
  • ЕСИА для юридически значимых действий — это 🔴: ГОСТ‑HTTPS, КриптоПро, КЭП и СМЭВ. «ЕСИА Смарт» / esia.pro — коммерческий шлюз стороннего вендора, а не сам ЕСИА; официальная точка интеграции — esia.gosuslugi.ru.
  • Пресет «Массовый портал (B2C)» показывает только то, что можно поставить кнопкой на чужой сайт. Внутренние логины маркетплейсов и сервисов (Wildberries, Магнит, Самокат, Яндекс Еда) в него не входят — вход через них сторонним сайтам недоступен.
